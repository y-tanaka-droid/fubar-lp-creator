## 1. 良いLPとは何か

### CVRの平均値と目標値

| 区分 | 平均CVR | 高パフォーマンス目標 |
|------|---------|------------------|
| 業界全体（リスティング） | 2〜3% | 5%以上 |
| BtoB（リスティング） | 3.04% | 6%以上 |
| BtoC（EC） | 1〜2% | 4%以上 |
| モバイル対応LP | 11.7% | — |

> **数値の使い方:** クライアントへの提案時、「現状のLPのCVRは業界平均2〜3%ですが、構成を見直すと5%以上を目指せます」という形で活用

### 高CVRに共通する3要素

1. **ファーストビューで3秒以内に価値を伝える**
   - ユーザーの60%以上がFVで離脱する
   - 「誰が」「何を」「どんな結果が得られるか」を3行以内で表現

2. **社会的証明の配置**
   - 口コミ・レビュー表示でCVR最大270%向上の事例あり
   - 数値付き実績が最も効果的（「月商300万→1億」「3ヶ月で申込数2倍」）

3. **CTAの設計と配置**
   - CTA改善だけで最大104.5%のCVR向上事例あり
   - FV・ページ中段・最下部の3箇所が基本

## 2. LP構成の黄金律（PASONA法）

### 新PASONA（2025年標準）

旧PASONAは「扇動（Agitation）」で不安を煽る手法だったが、現代ユーザーに嫌われる。
新PASONAは**共感（Affinity）**に変わり、押し付けないスタイルに進化。
※ FUBARの齋藤さんのトーン（「認識合ってますかね？」「〜できたらな」）と相性が良い。

| ステップ | 内容 | FUBAR向け実装例 |
|---------|------|----------------|
| **P**roblem | 顧客の悩みを明確化 | 「LINEを導入したのに全然反応がない…」 |
| **A**ffinity | 否定せず共感・寄り添う | 「それ、コンテンツの問題じゃなくて構造の問題なんです」 |
| **S**olution | 解決策の提示（数値・根拠付き） | 「5Lフレームで設計し直すと開封率が平均3倍になります」 |
| **O**ffer | 限定性・緊急性・特典 | 「今月限定で初月無料診断付き」 |
| **A**ction | 明確なCTA（ハードル低め） | 「まずは15分だけLINEで話しましょう」 |

### LP全体のセクション順序

```
① ファーストビュー（FV）
   - キャッチコピー（問題提起 or 結果提示）
   - サブコピー（誰向けか・何が変わるか）
   - CTA①（ハードル低め）

② 課題提示・共感
   - ターゲットが抱える悩みを3〜5点列挙
   - 「こんなこと思ってませんか？」系

③ 解決策・サービス説明
   - What（何ができるか）
   - How（どうやるか）
   - Why（なぜこれが効くか）

④ 実績・数値・事例
   - Before/After の数値
   - 導入事例 or 支援事例

⑤ 社会的証明（お客様の声）
   - 顔写真+名前+具体的なコメント

⑥ 料金・プラン説明（必要な場合）

⑦ FAQ

⑧ 最終CTA
   - 強めのコピー+行動ボタン

⑨ フッター（会社情報・プライバシーポリシー）
```

## 3. ファーストビューの設計原則

### FVで必ず入れる3要素

1. **ベネフィット or 数値実績**（「月商300万→1億にした〇〇」）
2. **ターゲット明示**（「年商1〜5億の中堅企業オーナー向け」）
3. **CTA**（「15分で無料相談」等、ハードル最小）

### FVのコピーパターン3選

**①結果先出し型（最強）**
```
外注30万円のLPが、AIで2時間でできる時代になりました。
→ 詳しく見る
```

**②問いかけ型**
```
LINEを使っているのに、毎月の売上に反映されていないですか？
→ 無料で原因を調べる
```

**③数値証明型**
```
200社のLINEマーケを支援。
平均開封率3倍・月商1億円達成事例あり。
→ まず事例を見る
```

## 4. CTA設計の原則

### CTAの配置場所（3点セット）

```
FV直下 → ページ中段（実績の後） → 最終CTA
```

### CTAコピーの書き方

**NG（硬い・ハードルが高い）**
- 「お問い合わせはこちら」
- 「購入する」
- 「申し込む」

**OK（柔らかい・ハードル低い）**
- 「まずはLINEで15分だけ話しましょう」
- 「無料でどこが問題か教えてもらう」
- 「事例だけ先に見てみる」

### LINEへの誘導CTAボタン（FUBAR専用）

```html
<a href="https://line.me/R/ti/p/[LINE_ID]" class="cta-button">
  <img src="line-icon.svg" alt="LINE">
  まずLINEで無料相談（15分）
</a>
```

スタイル指定:
```css
.cta-button {
  background: #06C755;  /* LINE グリーン */
  color: white;
  padding: 16px 32px;
  border-radius: 50px;
  font-size: 18px;
  font-weight: bold;
  display: inline-flex;
  align-items: center;
  gap: 8px;
  box-shadow: 0 4px 15px rgba(6, 199, 85, 0.4);
}
```

## 5. 業種別ベストプラクティス

### BtoB（FUBARのクライアント主体）

- **CV目標:** 資料請求・無料相談申込み（即購入ではない）
- **意思決定者:** 複数人が関わる → 論理と実績で説明できること
- **検討期間:** 数週間〜数ヶ月
- **LP特性:** ROI可視化、導入事例の詳細、比較表

**BtoB向けFV例:**
```
LINE活用で年間1,200万円の広告費を削減した事例があります。
→ 同業他社の事例を見る（無料）
```

### BtoC（ジム・クリニック・EC等）

- **CV目標:** 即購入・LINE登録・無料体験予約
- **意思決定:** 1人で即断
- **LP特性:** ビジュアル強め・感情訴求・ビフォーアフター

### サービス業（美容・教育・士業等）

- **CV目標:** 無料カウンセリング・初回体験予約
- **LP特性:** 雰囲気・信頼感・before-after

## 6. モバイルファースト設計

### 絶対に守るルール

| 項目 | 規則 |
|------|------|
| ブレークポイント | SP: 375px / TB: 768px / PC: 1280px |
| ボタンサイズ | 最小44×44px（タップミス防止） |
| フォントサイズ | 本文16px以上、見出し24px以上 |
| ページ読み込み | 3秒以内（1秒遅延ごとにCVR7%減） |
| CTAボタン | 常に画面下部に固定表示（スクロール中も見える） |

### 読み込み速度の改善ポイント

- 画像は WebP形式・最大800KB以下
- Google Fontsはdisplayオプションをswapに
- 不要なJSライブラリは削除
- 遅延ローディング（loading="lazy"）を画像に付与

## 7. CVR最適化の具体的施策

### 即効性が高い順ランキング

| 順位 | 施策 | CVR向上効果 |
|------|------|-----------|
| 1 | CTAボタンの文言変更 | 最大200% |
| 2 | 口コミ・レビューの追加 | 最大270% |
| 3 | FVコピーの書き直し | 50〜100% |
| 4 | ページ速度の改善 | 10〜30% |
| 5 | フォーム項目数を3つに絞る | 20〜40% |
| 6 | CTA3点配置（FV・中段・下部） | 30〜50% |

### ヒートマップで見るべき4ポイント

1. **スクロール率** — どこで読み込みが止まるか（=コンテンツの飽き点）
2. **クリックヒートマップ** — CTAが思ったよりクリックされていないか
3. **注視ヒートマップ** — FVの視線がどこに集まっているか
4. **モバイルタッチ** — SP環境でのタップ位置・誤操作

### フォーム最適化（EFO）

- **3項目が最高CVR（約25%）**
- BtoBでも「名前・会社名・メール」の3つまで
- それ以上は追加する前に「本当に必要か」を問う

## 8. AI制作ワークフロー（Claude Code版）

### 全体フロー

```
① GPT image 2 でデザインカンプ生成（20〜30分）
        ↓
② Claude でロゴ・素材の切り出し・背景透過（10〜15分）
        ↓
③ Claude Design（claude.ai）でLP本体をコード化（30〜40分）
        ↓
④ Claude Code（このCLI）で微修正・LINE URL差し替え（20〜30分）
        ↓
⑤ GitHub Pages / Vercel / FTP でデプロイ（10〜20分）

合計: 90〜135分
```

### STEP 1: GPT image 2 デザイン生成

**基本プロンプト構造:**
```
Create a professional landing page design for [サービス名].
Industry: [業種]
Target: [ターゲット層]
Style: [モダン/高級感/清潔感]
Color: [メインカラー]
Sections: hero, 3 features, testimonials, CTA
Language: Japanese text placeholders
Size: 1440x5000px (full page)
```

**FUBAR向けカスタマイズ:**
```
LINEマーケティングコンサルのLPを作成。
ターゲット: 年商1〜5億の中堅企業経営者
カラー: ネイビー（#1a2f5a）+ ゴールド（#d4af37）
雰囲気: 信頼感・実績重視・押し付けない
強調: 「LINE」「配信」「CVR」「実績数値」
```

### STEP 2: 素材の切り出し・透過

Claude（通常チャット）で:
```
この画像から[ロゴ/アイコン/人物素材]を切り出して
背景透過したPNG画像を生成してください。
```

### STEP 3: Claude Design でコード化

claude.ai のArtifacts（Designモード）で:
```
添付のデザイン画像を参考にLPをHTML+CSSで作成してください。

要件:
- レスポンシブ対応（375px / 768px / 1280px）
- フォント: Google Fonts（Noto Sans JP）
- アニメーション: スクロール時のfade-in（Intersection Observer）
- CTA: LINEへ誘導（URL: [LINE_URL]）
- セクション: FV / 課題 / 解決策 / 実績 / CTA / FAQ / フッター
- 単一HTML（CSS/JS込み）で出力
```

### STEP 4: Claude Code で仕上げ

このCLIで実行できる典型的な修正指示:

```bash
# SP表示崩れ修正
"SP（375px）でファーストビューのボタンが見切れている。修正して。"

# LINE URL一括差し替え
"CTAボタンのURL全箇所を https://line.me/R/ti/p/XXXX に変更して。"

# 固定CTAバナー追加
"スクロール中も常に画面下部に「LINEで無料相談」ボタンが固定表示されるように追加して。"

# OGP追加
"OGP設定を追加して。タイトル:[タイトル] 説明:[説明] 画像:[URL]"

# パフォーマンス改善
"全画像にloading=lazyを追加して。Google FontsのdisplayをswapにしてCore Web Vitalsを改善して。"
```

### STEP 5: デプロイ

**GitHub Pages（無料・推奨）:**
```bash
git init
git add .
git commit -m "LP公開: [クライアント名]"
gh repo create [repo-name] --public
git push -u origin main
# Settings > Pages > Deploy from main
```

**Vercel（独自ドメイン対応・推奨）:**
```bash
npm install -g vercel
vercel --prod
```

## 10. 技術スタック選定ガイド

| 用途 | 推奨スタック | 理由 |
|------|------------|------|
| シンプルLP・スピード重視 | HTML + CSS + Vanilla JS | ビルド不要・即デプロイ |
| SEO重視・高速表示 | Astro + Tailwind CSS | Zero JS・HTML-first |
| インタラクション豊富 | Next.js + Tailwind + Framer Motion | アニメーション強い |
| チーム開発・型安全 | Next.js + TypeScript + Tailwind | 再利用性高い |
| FUBARの案件（デフォルト） | HTML + CSS + Vanilla JS | 納品・FTP管理が簡単 |

### FUBAR案件推奨: 単一HTMLファイル構成

```
client_lp/
  index.html      ← CSS・JS全込みの単一ファイル
  images/
    hero.webp
    logo.svg
    client_photo.webp
  og_image.jpg    ← OGP用（1200×630px）
```

## 11. FUBAR向けLP制作チェックリスト

### クライアントから確認する事項

```
□ サービス名・キャッチコピー候補
□ ターゲット（年齢・職業・悩み・状況）
□ 強み・USP（競合との違い）
□ 実績・数値（Before/After形式で）
□ お客様の声（3件以上、顔写真があると尚良）
□ LINE登録URL / CTA先URL
□ 既存のブランドカラー・ロゴ（あれば）
□ 参考にしたいLPのURL（あれば）
□ 公開先（GitHub Pages / 独自ドメイン / 先方サーバー）
□ 納品形式（HTMLファイル / GitHubリポジトリ / Vercel URL）
```

### 制作後の品質確認

```
□ SP（375px）で表示崩れがないか
□ CTAボタンがFV・中段・下部の3箇所にあるか
□ LINE URLが全CTAに正しく設定されているか
□ 誤字・数値の確認（特に実績数値）
□ OGP設定が入っているか（SNSシェア時の見た目）
□ ページ読み込み3秒以内か（PageSpeed Insights確認）
□ HTTPS対応しているか（独自ドメインの場合）
□ プライバシーポリシーページ or リンクがあるか
```

### 5Lレイヤーとの対応

| LPの役割 | 5Lレイヤー | 主なCTA |
|---------|----------|--------|
| 広告→LP→LINE登録 | L1〜L2 | 友だち追加 |
| LINE→LP→成約 | L4 | 面談申込・購入 |
| LP→LP→LTV | L5 | アップセル・紹介 |
